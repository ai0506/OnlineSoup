# OnlineSoup 文档索引

本文件是仓库 Markdown 文档的导航地图，用来回答两个问题：

1. 当前任务应该先看哪份文档？
2. 某份文档负责记录什么，是否仍适合作为当前事实依据？

开始开发前建议先看 `AGENTS.md` 或 `CLAUDE.md`，再根据任务从本索引选择相关文档。文档内容与代码、迁移或实际数据库冲突时，以已验证的实现和数据库状态为准，并及时修正文档。

## 二、文档优先级

不同文档发生冲突时，建议按以下顺序判断：

1. 用户本次明确要求。
2. `AGENTS.md` 或 `CLAUDE.md` 中的当前协作和安全规则。
3. 已部署数据库、最新迁移、当前代码和真实测试结果。
4. `tasks.md` 中维护的当前进度。
5. `多人项目聊天室房间设计.md` 中的核心产品规则。
6. `多人在线海龟汤聊天室开发设计.md`、`README.md` 和 `本地部署教程.md` 中的早期或说明性内容。

可以把它理解为：设计文档是“建筑图纸”，代码和数据库是“实际建成的房子”。两者不一致时，先查清房子的真实状态，再决定修代码还是更新图纸。

## 四、常见任务的推荐阅读顺序

| 任务 | 推荐顺序 |
| --- | --- |
| 第一次运行项目 | `README.md` → `本地部署教程.md` → `.env.example` |
| 查看当前进度 | `tasks.md` → `updates.md` → Git 历史 |
| 修改房间、座位或积分 | `AGENTS.md`/`CLAUDE.md` → `多人项目聊天室房间设计.md` → 相关迁移与代码 |
| 修改聊天或 Realtime | `AGENTS.md`/`CLAUDE.md` 的“聊天与 Realtime” → 相关组件、Route 和 RPC |
| 接入题库和 AI 主持 | `tasks.md` 阶段六 → `prompts_suggestion.md` → `多人在线海龟汤聊天室开发设计.md` 对应章节 |
| 修改 Supabase 权限或迁移 | `AGENTS.md`/`CLAUDE.md` → Supabase `SKILL.md` → 相关 Postgres 最佳实践 → 当前迁移 |
| 排查数据库性能 | Postgres `SKILL.md` → 对应 `query-`/`conn-`/`monitor-` 规则 → `EXPLAIN ANALYZE` 实测 |
| 查看 AI 助手做过什么 | `updates.md`；需要精确差异时再看 Git diff/log |
